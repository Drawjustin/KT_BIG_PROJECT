Telecom 기술 선택 배경

서비스 플로우

모바일(안드로이드,아이폰)이 특정번호로 전화 -> 컴퓨터가 이를 수신 -> AI 서버와 통신하며 통화내용에 따라, 특정 기능을 실행


기술 선정 배경
현재 전화통신의 경우
PSTN과 VoIP라는 두가지 기술이 존재함

PSTN -> 아날로그 전화망, 전통적인 회선 교환 방식 // 다만, 현재 PSTN은 디지털 회선 교환 기술도 포함하며, 교환기 간 통신은 대부분 디지털 기반으로 처리
VoIP -> 디지털 전화망, 인터넷 회선 교환 방식 // SIP(Session Initiation Protocol)

현재 모바일 대부분의 경우 전화 기본 앱은 PSTN 방식을 사용함

즉 아날로그 -> 디지털로의 전환이 필요함.
PSTN + VoIP 둘다 사용해야함. 

PSTN -> VoIP 변환에 필요한 서비스
1. 전화번호 -> VoIP 서버와 연동해주는 서비스 (SIP 트렁크 서비스)
※실시간으로 PSTN 신호를 디지털화 하기 위해서는 하드웨어 장비가 필요함※
-> 현재로서는 서비스를 이용하는 것이 최선
대표 서비스: Twilio, Plivo, Nexmo.

2. VoIP 서버(직접구축)
VoIP 신호를 관리하고, 전화 이벤트를 컴퓨터로 전달.
대표 서버 소프트웨어: Asterisk, FreeSWITCH, Kamailio, OpenSIPS.


PSTN : 음성(아날로그 신호)를 전기 신호로 바꾸어서 유선상(전화 망) 전송하고, 이를 하드웨어로 해석해서 다시 음성으로 듣는 것
VoIP : 음성을 패킷으로 바꾸어서 유선상(네트워크 망) 전송하고, 이를 다시 소프트웨어로 해석해서 다시 음성으로 듣는것 

쉽게말해서 VoIP와 PSTN방식은 알고보면 똗같은데, 과거 PSTN방식이 유선상에서 기술의 발전에 따라 
무선 주파수를 통해 핸드폰-기지국-PSTN 전화망 - 상대근처 기지국 - 핸드폰 방식으로 발전되고, 
이때 데이터 형식이 패킷이냐, PCM이냐에 따라, 인터넷 네트워크인지, 전화망 네트워크인지만 좀 다르다.
